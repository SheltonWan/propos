# PropOS Phase 1 实施清单 v1.7

> 版本: v1.2
> 日期: 2026-04-09
> 依据文档: PRD v1.7 / ARCH v1.2 / data_model v1.3
> 目标: 将 Phase 1 拆解为可执行的实施任务，按 Must / Should 分级推进。

---

## 一、实施原则

1. 优先保证六条核心链路闭环：资产、合同、账单、收款、工单、二房东填报。
2. Must 项未完成前，不以 Should 项替代核心交付。
3. 所有涉及财务、安全、审核、审计的能力，默认优先于可视化增强。
4. 所有批量导入、定时任务、外部门户能力必须有失败补偿设计。
5. v1.7 新增的押金管理、水电抄表、营业额申报、导入批次管理纳入 Must，因其直接构成业财闭环。

## 五、推荐实施顺序

### 第 1 批

M-01 ~ M-08，先完成底座、台账、导入、图纸基础能力。

### 第 2 批

M-09 ~ M-16b，完成合同（含多单元/终止/押金）、账单（含水电/分成）、收款、NOI 主链路。

### 第 3 批

M-17 ~ M-23，完成工单与二房东穿透两条运营链路。

### 第 4 批

M-24a ~ M-27，加固导入治理、安全、审计、验收闭环，并并行推进 Should 项。

### 第 5 批

M-28 → M-29 → M-30 → M-31 → M-32 → M-33，先建组织架构，再完成 KPI 正式考核全部能力（方案配置、评分、排名、申诉、导出）。

## 七、交付提醒

1. Must 项中任何涉及财务口径的需求，不得用前端展示逻辑替代后端规则实现。
2. 二房东门户必须在 RBAC 之外叠加 Repository 层数据隔离，不接受仅靠前端隐藏。
3. 所有 PDF、导入模板、样本数据都应纳入验收交付物，而不是口头说明。
4. 押金管理不计入 NOI 收入（v1.7 明确），禁止将押金混入应收账单体系。
5. WALE 计算需同时输出双口径（收入加权 + 面积加权），已终止合同剩余租期归零。
6. NOI 统一使用不含税口径，账单需同时展示含税与不含税金额。
