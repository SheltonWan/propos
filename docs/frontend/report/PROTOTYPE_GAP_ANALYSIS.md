## 一、原型定位与技术上下文

### 1.1 原型技术栈

`frontend/` 是一个独立的 **React 19 + TypeScript + TailwindCSS + react-router v7 + recharts** 高保真原型，用于在浏览器中预览移动端交互设计，**非生产代码**。实际交付分为：

| 端 | 项目目录 | 技术栈 | 用途 |
|----|---------|--------|------|
| uni-app 移动端 | `app/` | Vue 3 + wot-design-uni + Pinia | iOS / Android / HarmonyOS / 微信小程序 / H5 |
| Admin PC 端 | `admin/` | Vue 3 + Element Plus + Pinia | 管理层 PC 操作 |
| 原型 | `frontend/` | React 19 + TailwindCSS | 设计验证 / 产品评审（不上线） |

## 二、一级导航结构对比

### 2.1 TabBar 对比

| 序号 | PAGE_SPEC v1.8 | 原型实际 | 匹配 | 差距说明 |
|------|---------------|---------|------|---------|
| 1 | **总览** `pages/dashboard/index` | **首页** `/` (Home.tsx) | ✅ | 文案从"首页"调整为"总览"即可 |
| 2 | **资产** `pages/assets/index` | **资产** `/assets` (Assets.tsx) | ✅ | 完全匹配 |
| 3 | **合同** `pages/contracts/index` | **合同** `/contracts` (Contracts.tsx) | ✅ | 完全匹配 |
| 4 | **工单** `pages/workorders/index` | **工单** `/work-orders` (WorkOrders.tsx) | ✅ | 完全匹配 |
| 5 | **财务** `pages/finance/index` | **我的** `/profile` (Profile.tsx) | ❌ | **结构性偏差：财务一级入口被「我的」替代** |

### 2.2 核心问题

第 5 个 Tab 位被「我的」（用户个人中心）占据，**财务模块作为一级入口完全缺失**。财务是 PRD Module 3（Must 优先级）的核心入口，涵盖账单管理、收款核销、NOI 看板、KPI 考核等高频操作。

**建议**：将第 5 Tab 从「我的」改为「财务」，「我的」改为首页右上角头像入口（L2 页面）。

## 五、页面层级调整建议

### 5.1 一级页面（TabBar）调整

```
原型现状:   首页     资产     合同     工单     我的
            ↓        ↓        ↓        ↓        ↓
建议调整:   总览     资产     合同     工单     财务
```

- 「我的」→ 降级为 L2 页面，入口改为总览页右上角头像
- 「财务」→ 升级为第 5 Tab，成为 M3 财务模块一级入口

以下功能点应直接在现有 L1 页面中补充，不需要新增页面：

## 六、优先级排序建议

基于 PRD Must/Should/Could 分级和实际开发依赖关系，建议按以下顺序补齐：

### P0 — 立即调整（结构性问题）

1. **TabBar 第 5 位从「我的」改为「财务」**
2. **新增财务一级页面** `pages/finance/index`
3. **「我的」降级为 L2**，入口移至总览页右上角

### P1 — 高优先级新增页面

4. 合同详情页 `pages/contracts/detail`（从内嵌 Panel 拆出）
5. 账单列表页 `pages/finance/invoices`
6. 工单详情页 `pages/workorders/detail`（从内嵌展示拆出）
7. 单元详情页 `pages/assets/unit-detail`（从弹出面板拆出）

### P2 — 中优先级新增页面

8. KPI 考核页 `pages/finance/kpi`
9. 二房东管理列表 `pages/subleases/index`
10. 租客画像页 `pages/contracts/tenant`

### P3 — 低优先级（可随开发迭代补充）

11. 二房东详情 `pages/subleases/detail`
12. 账单详情 / 核销 `pages/finance/invoice-detail`
13. 新建工单独立页 `pages/workorders/new`

## 七、原型亮点总结

尽管覆盖度不足，原型在以下方面的设计质量值得肯定，可直接作为 uni-app 实现的设计参照：

| 亮点 | 设计质量 | 可复用程度 |
|------|---------|-----------|
| SVG 热区平面图 + 三种图层模式 | ★★★★★ | 高 — 交互模型可直接迁移 |
| 合同模板系统 + 6 种递增类型配置器 | ★★★★★ | 高 — 表单结构完整 |
| NOI 仪表盘（业态分拆/趋势/预算/楼栋下钻） | ★★★★★ | 高 — 数据维度完整 |
| WALE 仪表盘（瀑布图/到期风险/趋势） | ★★★★★ | 高 — 创新性展示形式 |
| 三类工单完整流程 | ★★★★☆ | 高 — 表单 + 列表 + 完工结构清晰 |
| 色彩语义一致性 | ★★★★☆ | 高 — 符合 PRD 状态色规范 |
| Mobile-first 布局 + 安全区适配 | ★★★★☆ | 中 — 需迁移至 uni-app 布局体系 |

---

## 八、结论

原型在**分析型页面**（NOI、WALE、热区图）上的设计深度远超预期，在**操作型页面**（财务日常操作、租客管理、二房东管理）上存在显著空白。最关键的结构性问题是**财务一级入口缺失**，需立即修正导航架构。

**总体建议**：以当前原型为基础，优先完成 P0 导航结构调整和 P1 核心详情页设计，即可覆盖 PRD Phase 1 约 80% 的移动端页面需求。Admin PC 端的管理型操作（批量导入/续签/终止/押金管理/审计日志等）不需要在移动原型中体现。
