# PropOS 产品需求文档摘要

> **项目代号**: PropOS（Property Operating System）
> **原文版本**: v1.8　**更新日期**: 2026-04-09
> **摘要生成日期**: 2026-04-14

---

## 一、项目概述

PropOS 是为自有混合型物业构建的内部数字化资产运营管理平台，将资产状态、租务合同、财务收支与物业工单四大核心业务统一纳入系统管理。

| 指标 | 数据 |
|-----|------|
| 总建筑面积（GFA） | 约 40,000 m² |
| 总房源数量 | 639 套（写字楼 + 商铺 + 公寓） |
| A 座建面 | 约 30,000 m²（整栋写字楼） |
| 商铺建面 | 约 2,707 m² |
| 业态占比 | 写字楼 69% / 公寓 27% / 商铺 4% |

### 三业态核心差异

| 维度 | 写字楼 | 商铺 | 公寓 |
|-----|--------|------|------|
| 合同周期 | 1~3 年 | 3~5 年 | 6~12 个月 |
| 租金计算 | 元/m²/月 | 固定租金 ± 营业额分成 | 整套月租金 |
| 免租期 | 1~2 周 | 1~3 个月 | 无 |

| 角色 | 标识 | 主要权限 |

| 超级管理员 | `super_admin` | 全数据访问、系统配置、用户管理 |

| 维修技工 | `maintenance_staff` | 工单接派处理、水电抄表录入 |

## 三、Phase 1 功能模块

### 范围分级

| 分级 | 核心内容 |
|------|---------|
| **Must（必交）** | 资产台账、CAD 展示、合同录入与状态机、租金递增规则、自动账单、收款核销、基础 NOI、工单闭环、二房东填报与审核、审计日志、KPI 正式考核 |
| **Should（高价值）** | 资产概览看板、WALE 趋势、催收模板、楼栋/业态下钻报表、穿透出租率 |
| **Could（预留）** | 二房东 API 自动推送、复杂溢价分析、消息通道扩展 |

---

### 模块 1：资产与空间可视化

- 楼栋 → 楼层 → 单元三级数字台账
- 支持 .dwg 转 SVG/PNG，楼层热区状态色块（已租/即将到期/空置/非可租）
- 业态差异化字段（写字楼：工位数；商铺：门面宽/临街面；公寓：卧室数）
- 改造记录与照片管理
- 资产台账 Excel 导出（639 套）

---

### 模块 2：租务与合同生命周期管理

#### 租客全景画像

- 基本信息、租赁历史、工单记录
- **信用评级**（系统内部自动计算，每月 1 日重算）：

| 评级 | 规则 |
|------|------|
| A（优质） | 过去 12 个月逾期 ≤ 1 次，且单次 ≤ 3 天 |
| B（一般） | 逾期 2~3 次，或单次 4~15 天；新租户默认 B |
| C（风险） | 逾期 ≥ 4 次，或单次 > 15 天 |
| D（严重违约） | 逾期 ≥ 6 次，或单次 > 30 天 |

> 新租户默认 B 级，签约满 3 个月后首次评级。

#### 合同管理

- 多单元绑定、合同状态机（报价中 → 待签约 → 执行中 → 即将到期 → 已到期/续签/终止）
- 支持正常到期、提前退租、协商终止、业主单方解约四类终止路径
- 押金独立建账（收取 → 冻结 → 冲抵 → 退还），不计入 NOI

#### 智能预警引擎

| 预警类型 | 触发时机 |
|---------|---------|
| 租约到期预警 | 提前 90 / 60 / 30 天 |
| 租金逾期预警 | 账期后第 1 / 7 / 15 天未到账 |
| 月度到期汇总 | 每月 1 日推送 |
| 押金退还提醒 | 合同终止前 7 天 |

#### WALE 计算（加权平均租期到期年数）

$$WALE_{\text{收入}} = \frac{\sum (剩余租期_i \times 年化合同租金_i)}{\sum 年化合同租金_i}$$

$$WALE_{\text{面积}} = \frac{\sum (剩余租期_i \times 计费面积_i)}{\sum 计费面积_i}$$

- 支持组合级、楼栋级、业态级三个维度
- 剩余租期精确到天（单位：年）

#### 租金递增规则配置器

支持 6 种递增类型：固定比例、固定金额、阶梯式、CPI 挂钩、每 N 年递增、免租后基准调整，支持多阶段混合配置与模板复用。

### 模块 3：财务与业财一体化

#### 账单自动生成

- 依据合同条款自动生成应收账单（租金、物业管理费、水电代收、停车费等）
- 支持部分收款、跨账单核销，保留核销轨迹

#### NOI 实时看板

$$NOI = EGI - OpEx \quad;\quad EGI = PGI - 空置损失 + 其他收入$$

| 展示层级 | 内容 |
|---------|------|
| L1 概览（首页） | 当月 NOI、NOI Margin、出租率、收款率 |
| L2 看板（财务模块）| 业态分拆、支出类目、历史趋势、楼栋下钻、预算对比 |
| L3 明细（下钻）| 逐笔明细、未缴款租户、空置单元列表 |

#### KPI 考核仪表盘

| 指标编号 | 名称 | 满分标准 |
|---------|------|---------|
| K01 | 出租率 | ≥95% |
| K02 | 收款及时率 | ≥95% |
| K03 | 租户集中度（反向） | ≤40% |
| K04 | 续约率 | ≥80% |
| K05 | 工单响应时效（反向） | ≤24h |
| K06 | 空置周转天数（反向） | ≤30 天 |
| K07 | NOI 达成率 | ≥100% |
| K08 | 逾期率（反向） | ≤5% |
| K09 | 租金递增执行率 | ≥95% |
| K10 | 租户满意度（手动） | ≥90 分 |

$$KPI_{总分} = \sum_{i=1}^{n} (指标得分_i \times 权重_i)$$

- 支持多套方案并存，评估周期可选月度/季度/年度
- 快照冻结后 7 日内可提交申诉，申诉需运营管理层审批

### 模块 4：物业运营与工单系统

#### 工单类型

| 类型 | 场景 | 成本归集 |
|------|------|---------|
| `repair` 报修 | 设备故障、设施损坏 | OpEx 汇入 NOI；CapEx 不计入 |
| `complaint` 投诉 | 服务投诉、环境问题 | 无 |
| `inspection` 退租验房 | 合同到期/提前退租查验 | 无（产生的维修工单有成本） |

#### 双端支持

- **Flutter App 包（主力）**：全工单流程 + 推送 + CAD 快查 + 扫码报修
- **微信小程序（精简版）**：仅扫码报修 + 查看状态

#### 派单 SLA

| 场景 | 时限 |
|------|------|
| 一般报修 | 4 小时内派单 |
| 紧急报修 | 30 分钟内派单 |
| 非常紧急 | 10 分钟内派单 |
| 投诉 | 2 小时内派单 |

---

### 模块 5：二房东租赁信息穿透管理

- 目标：穿透主合同层级，掌握二房东转租终端租客的真实信息
- **双通道采集**：内部手工录入 / 二房东外部自助 Web 填报
- 数据流：填报 → 待审核 → 审核通过 → 参与穿透看板计算
- 穿透分析：转租溢价率、穿透出租率、楼层穿透视图
- 安全：账号与主合同绑定，到期自动冻结；连续 5 次登录失败锁定 30 分钟；强制 HTTPS

## 四、非功能性要求摘要

| 类别 | 要求 |
|-----|------|
| 访问端 | PC Web 管理后台（Element Plus） + Flutter 移动端 + 微信小程序精简版 |
| 性能 | 仪表盘 < 3 秒；50 并发用户；账单批量生成 639 条 < 30 秒 |
| 安全 | 证件号加密存储，角色权限严格隔离，PIPL 合规 |
| 审计日志 | 合同修改、账单核销、权限变更、二房东操作均留日志 |
| 备份 | 每日自动备份，RPO ≤ 24h，RTO ≤ 4h |
| 语言 | Phase 1 仅简体中文 |
