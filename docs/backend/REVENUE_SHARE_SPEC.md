# PropOS 商铺营业额分成计算详解与边界案例

> **版本**: v1.0  
> **日期**: 2026-04-08  
> **依据**: PRD v1.7（2.2.3 商铺营业额分成对账机制）/ API_CONTRACT v1.7  
> **用途**: 为 AI 实现 `revenue_share` 计算逻辑提供完整数值示例与边界处理规则  

---

## 一、核心计算公式

$$当月应收 = MAX(保底月租金,\ 月营业额 \times 分成比例)$$

其中：
- **保底月租金** = 合同约定的当期基准月租金（受递增规则影响，可逐期调整）
- **分成比例** = 合同约定的营业额提成百分比（如 8%）

---

## 二、完整计算示例

### 场景：聚鑫餐饮（合同 C-RETAIL-01）

| 合同参数 | 值 |
|---------|-----|
| 单元 | S101（计费面积 108 m²） |
| 合同期 | 2025-03-01 ~ 2028-02-28 |
| 免租期 | 2025-03 ~ 2025-05（3个月） |
| 保底单价 | 阶梯式（见下表） |
| 分成比例 | 8% |
| 税率 | 5%（简易征收） |

**保底租金递增表**：

| 阶段 | 区间 | 保底单价 | 保底月租(不含税) |
|------|------|---------|----------------|
| 1 | 2025-06 ~ 2026-02 | ¥230/m² | 108 × 230 = ¥24,840 |
| 2 | 2026-03 ~ 2027-02 | ¥250/m² | 108 × 250 = ¥27,000 |
| 3 | 2027-03 ~ 2028-02 | ¥270/m² | 108 × 270 = ¥29,160 |

### 逐月计算（2025年6月~12月）

| 月份 | 月营业额 | 8%分成额 | 保底月租 | 应收(不含税) | 取值来源 | 含税应收 | 差额 |
|------|---------|---------|---------|------------|---------|---------|------|
| 2025-06 | ¥250,000 | ¥20,000 | ¥24,840 | **¥24,840** | 保底 | ¥26,082.00 | — |
| 2025-07 | ¥350,000 | ¥28,000 | ¥24,840 | **¥28,000** | 分成 | ¥29,400.00 | +¥3,160 |
| 2025-08 | ¥310,000 | ¥24,800 | ¥24,840 | **¥24,840** | 保底 | ¥26,082.00 | — |
| 2025-09 | ¥500,000 | ¥40,000 | ¥24,840 | **¥40,000** | 分成 | ¥42,000.00 | +¥15,160 |
| 2025-10 | ¥180,000 | ¥14,400 | ¥24,840 | **¥24,840** | 保底 | ¥26,082.00 | — |
| 2025-11 | ¥310,500 | ¥24,840 | ¥24,840 | **¥24,840** | 恰好相等→保底 | ¥26,082.00 | — |
| 2025-12 | ¥620,000 | ¥49,600 | ¥24,840 | **¥49,600** | 分成 | ¥52,080.00 | +¥24,760 |

### 递增后的计算（2026年3月阶段2生效）

| 月份 | 月营业额 | 8%分成额 | 保底月租 | 应收(不含税) | 取值来源 |
|------|---------|---------|---------|------------|---------|
| 2026-03 | ¥350,000 | ¥28,000 | ¥27,000 | **¥28,000** | 分成 |
| 2026-04 | ¥300,000 | ¥24,000 | ¥27,000 | **¥27,000** | 保底 |

> **注意**: 递增后保底线上升到 ¥27,000，相同营业额 ¥350,000 仍触发分成（28,000 > 27,000），但 ¥300,000 的分成额（24,000）已低于新保底线。

## 三、边界案例

### 3.1 免租期月份

| 月份 | 营业额 | 处理 |
|------|-------|------|
| 2025-03~05 | 不适用 | 免租月不收租金，账单标记 `exempt`；**免租期内不要求商户申报营业额** |

### 3.2 分成额与保底完全相等

```
月营业额 = ¥310,500，8% = ¥24,840.00
保底月租 = ¥24,840.00
MAX(24,840, 24,840) = ¥24,840（取保底）
```

系统实现应为 `>=` 判断：`revenue_share >= base_rent` 时取 `revenue_share`，否则取 `base_rent`。相等时取哪个不影响金额，但计费来源标记为 `base_rent`。

### 3.3 商户迟报 / 未申报

| 场景 | 处理 |
|------|------|
| 当月未申报 | 先按保底租金生成账单（`item_type: rent`），待商户补报后重算差额 |
| 差额处理 | 补报后 `分成额 > 保底` 时，生成差额补充账单（`item_type: revenue_share`），金额 = 分成额 - 保底 |
| 补报后无差额 | 不生成补充账单 |

### 3.4 补报修正（已申报月份修改营业额）

```
原申报: 2025-07 营业额 ¥350,000 → 应收 ¥28,000（已出账）
修正: 2025-07 营业额改为 ¥400,000 → 应收应为 ¥32,000
差额: ¥32,000 - ¥28,000 = ¥4,000
```

系统处理：
1. 原账单金额不变（已出账或已核销的不可修改金额）
2. 生成差额调整账单：`¥4,000`（不含税），`item_type: revenue_share`，备注关联原账单
3. 审计日志记录修正原因

### 3.5 修正导致退款（实际营业额低于原申报）

```
原申报: 2025-09 营业额 ¥500,000 → 应收 ¥40,000（已核销）
修正: 2025-09 营业额改为 ¥200,000 → 应收应为 ¥24,840（保底）
差额: ¥24,840 - ¥40,000 = -¥15,160
```

系统处理：
1. 生成负数调整账单（信用单）：`-¥15,160`
2. 可用于抵扣后续月份账单，或与下期账单合并核销
3. 需财务主管审批（金额为负需额外确认）

## 四、税费处理规则

| 项目 | 规则 |
|------|------|
| 计算口径 | 分成额计算全程使用**不含税**口径 |
| 含税账单 | 最终账单含税金额 = 不含税应收 × (1 + tax_rate) |
| NOI 计入 | NOI 使用不含税金额 |
| 发票开具 | 按含税金额开票，税额 = 含税金额 - 不含税金额 |

**税率适用**:
- 一般纳税人出租不动产：增值税 9%
- 简易征收（小规模/老项目）：5%
- 合同中明确约定适用税率，系统按合同 `tax_rate` 字段计算

### 税费计算示例（2025年7月，简易征收5%）

```
不含税应收 = MAX(24,840, 28,000) = ¥28,000.00
税额       = 28,000 × 5% = ¥1,400.00
含税应收   = 28,000 + 1,400 = ¥29,400.00
```

## 五、账单生成逻辑伪代码

```dart
/// 计算商铺当月应收（不含税）
Decimal calculateRetailMonthlyRent({
  required Decimal baseRent,         // 当期保底月租（不含税）
  required Decimal shareRate,        // 分成比例（如 0.08）
  required Decimal? monthlyRevenue,  // 商户申报月营业额（可能为 null）
}) {
  if (monthlyRevenue == null) {
    // 未申报时按保底出账
    return baseRent;
  }
  final shareAmount = monthlyRevenue * shareRate;
  return shareAmount >= baseRent ? shareAmount : baseRent;
}
```

## 六、与生成 WALE 的关联

商铺分成合同在 WALE 计算中的年化租金取**保底租金**口径（不含分成部分），因为：
1. 分成额具有不确定性，不应纳入 WALE 基础加权
2. 保底租金反映合同的最低租金义务（minimum lease commitment）
3. 符合 MSCI/IPD WALE 计算通行做法

```
年化租金 = 保底月租 × 12（不含分成收入）
```
