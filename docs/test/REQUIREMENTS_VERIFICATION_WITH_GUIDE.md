## 使用说明

1. 每个需求点对应 PRD 中一个具体功能或约束，粒度以"可独立验证"为准
2. 验收时通过**实际测试** + **代码审查**双重确认，二选一不够
3. 标注 🔄 时须在括号中补注缺失部分
4. 本清单不替代单元测试，两者互补；核心计算类额外依赖自动化测试覆盖
5. 标注 ✅ 的需求附带 **「验证指引」** 小节，参照 GAP_VERIFICATION_GUIDE 格式提供操作步骤和预期结果
